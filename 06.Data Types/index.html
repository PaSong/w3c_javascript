JavaScript 데이터 유형

JavaScript 변수는 숫자,문자열, 객체 등 많은 데이터 유형을 보유 할 수 있느

숫자와 문자열을 추가 할 때 JavaScript는 숫자를 문자열로 처리합니다

JavaScript는 동적 유형이 있습니다.즉,동일한 변수를 사용하여 다른 데이터 유형을 유지할 수 있습니다.

typeof 연산자는 javaScript 배열이 객체이기 때문에 배열에 대해 "object"를 반환합니다

자바 스크립트 불리언

부울은 참 또는 거짓의 두가지 값만 가질 수 있음

bool은 조건부 테스트에 사용됨.

자바 스크립트 배열

배열은 []대괄호로 작성됨

배열 항목은 쉼표로 구분

배열 인덱스는 0부터 시작함.

자바 스크립트 객체

객체는 중괄호로 작성됨

객체 속성은 쉽표로 구분 된 이름: 값

var person ={firstName:'John', lastName:'Doe',age:50,eyeColor:'blue'}

연산자 유형

typeof 연산자를 사용하여 자바 스크립트 변수의 유형을 찾을 수 있음

정의되지 않음

var car;(정의되지 않습니다.)

undefined로 설정하면 변수를 비울 수 있습니다.(정의되지 않습니다.)

빈값

var car=""

빈 값은 undefined와 아무런 관련이 없음

null

null은 값을 존재하지 않게 만듬

JavaScript에서는 null의 데이터 유형이 객체

typeof null이 객체라는 점을 자바 스크립트의 버그라고 생각할 수 있음.

정의되지 않음과 널 (null) 사이의 차이점

typeof undefined // undefined

typeof ull // object

데이터형태가 다름

복잡한 데이터

typeof 연산자는 두 개의 복합 유형 중 하나를 반환 할 수 있음

typeof 연산자는 함수에 대해 객체를 반환하지 않음

typeof 연산자는 자바스크립트 배열이 객체이기 때문에 배열에 대해 object를 반환
