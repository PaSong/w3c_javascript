스크립트 유형 변환

Number()는 Number로 변환되고, String()은 String으로 변환되며, Boolean()은 부울로 변환됨

데이터 유형

자바스크립트에는 값을 포함 할 수 있는 5가지 데이터 유형이 있음.

  • String
  • Number
  • Boolean
  • object
  • function

3가지 유형의 객체가 있음.

  • object
  • Date
  • Array

값을 포함 할 수 없는 2가지 데이터 타입

  • null
  • undefined

연산자 유형

typeof연산자를 사용하여 JavaScript변수의 데이터 유형을 찾을 수 있음.

document.getElementById("demo").innerHTML= typeof "john" + "
" + typeof 3.14 + "
" + typeof NaN + "
" + typeof false + "
" + typeof [1,2,3,4] + "
" + typeof {name : 'john', age:34} + "
" + typeof new Date() + "
"+ typeof function () {} + "
" + typeof myCar + "
"+ typeof null;

  • NaN의 데이터 유형은 숫자임.
  • 배열의 데이터 유형은 object임.
  • 날짜의 데이터 유형은 객체
  • null의 데이터 형은 object임.
  • 정의되지 않은 변수의 데이터 유형이 undefined
  • 값을 할당받지 않은 변수의 데이터 유형도 undefined
typeof를 사용하여 자바스크립트 객체가 배열(또는 날짜)인지 판단 할 수 없음.

형식의 데이터 형식

typeof 연산자는 변수가 아님. 운영자임. 연산자(+ - * /)에는 데이터 유형이 없음.

그러나 typeof연산자는 항상 문자열(피연산자 유형 포함)을 반환.

생성자 속성

생성자 속성은 모든 자바 스크립트 변수의 생성자 함수를 반환

document.getElementById("demo").innerHTML = "john".constructor + "
"+ (3.14).constructor + "
"+ false.constructor + "
"+ [1,2,3,4].constructor + "
"+ {name:'john',age:34}.constructor + "
"+ new Date().constrounctor + "
" + function () {}.constructor;

생성자 속성을 검사하여 객체가 Array인지 여부를 확인할 수 있음.(Array라는 단어 포함)

var fruits =["banana","orange","apple","mango"]; document.getElemenetById("demo").innerHTML = isArray(fruits); function isArray(myArray){ return myArray.constructor.toString().indexOf("Array") > -1; }

객체가 Array 함수인지 쉽게 확인 할 수 있음

var fruits =["banana","orange","apple","mango"];

document.getELementById("demo"),innerHTML = isArray(fruits);

function isArray(myArray){ return myArray.constructor === array; }

자바 스크립트 유형 변환

자바스크립트 변수는 새 변수 및 다른 데이터 유형으로 변환 할 수 있음

  • 자바 스크립트 함수 사용
  • 자바스크립트 자체에 의해 자동으로

숫자를 문자열로 변환

전역 메소드 String()은 숫자를 문자열로 변환 할 수 있음.

숫자,리터럴,변수 또는 표현식의 모든 유형에서 사용할 수 있음.

var x = 123; document.getElementById("demo").innerHTML = String(x) + "
"+ String(123) + "
"+ String(100+23);

Number 메소드 toString()도 동일함.

var x =123;

document.getElementById("demo").innerHTML = x.toString() + "
" + (123).toString + "
" + (100+23).toString();

부울을 문자열로 변환

전역 메소드 String()은 부울을 문자열로 변환 할 수 있음

String(false) //return "false"

String(true) //return "true"

toString()도 마찬가지

false.toString() // return "false"

true.toString() //return "true"

날짜를 문자열로 변환

String(Date())

toString()도 동일

Date().toString()

문자열을 숫자로 변환

전역 메서드인 Number()은 문자열을 숫자로 변환 할 수 있음.

빈 문자열은 0으로 변환

다른 것은 NaN(숫자가 아님)으로 변환

  • Number("3.14") // return 3.14
  • Number(" ") //return 0
  • Number("") //return 0
  • Number("99 88") //return NaN

단항 + 연산자

단항 + 연산자는 숫자에 변수를 변환 할 수 있음

function myFunction(){ var y ="5"; var x = +y; document.getElementById("demo").innerHTML = typeof y + "
" + typeof x; }

부울을 숫자로 변환

전역 메서드 인 Number()는 부울을 숫자로 변환 할 수도 있음.

Number(false) //return 0

number(true) //return 1

날짜를 숫자로 변환

전역 메서드 Number()는 날짜를 숫자로 변환하는데 사용할 수 있음.

d = new Date()

Number(d) //returns 1404568027739

date 메소드 getTime()도 동일

d = new Date();

d.getTime() //returns 1404568027739

자동유형변환

자바스크립트가 "잘못된" 데이터 유형으로 작동하려고 하면 값을 "올바른" 유형으로 변환하려고 시도.

하지만 항상 결과가 옳은것은 아님

자동 문자열 변환

자바스크립트는 객체 또는 변수를 "출력"하려고하면 자동으로 변수의 toString() 함수를 호출함.

document.getElementById("demo").innerHTML = myVar;

숫자와 불린도 변환되지만 이것은 잘 보이지 않음.
