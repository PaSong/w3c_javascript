자바 스크립트 객체

JavaScript에서는 객체가 왕이됨. 객체를 이애하면 자바 스크립트를 이해하게 됨

JavaScript에서는 거의 모든 것이 "객체"임

  • 부울은 객체 일 수 있음(new 키워드로 정의 된 경우)
  • 숫자는 객체가 될 수 있음(new 키워드로 정의 된 경우)
  • 문자열은 객체가 될 수 있음(new 키워드로 정의 된 경우)
  • 날짜는 항상 객체
  • 수학은 항상 객체
  • 정규 표현식은 항상 객체
  • 배열은 항상 객체
  • 함수는 항상 객체
  • 객체는 항상 객체

프리미티브를 제외한 모든 JavaScript값은 객체임

자바 스크립트 기본 요소

프리미티브 값은 어떤 특성 또는 방법이 없는 값임.

기본 데이터 형식은 원기 값을 갖는 데이터임.

  • string
  • number
  • boolean
  • null
  • undefined

원시적 치는 불변임(하드 코드외어 있어 변경할 수 업음)

x =3.14이면 x의 값을 변경할 수 있음. 그러나 3.14의 값은 변경할 수 없음

객체는 변수임

JavaScript변수는 단일 값을 포함 할 수 있음

객체도 변수임. 그러나 객체에는 많은 값이 포함될 수 있음

var person = {firstName:"John", lastName:"Doe", age:50, eyeColor:"blue"};

자바스크립트 객체는 명명된 값의 모음

객체 속성

JavaScript 객체의 명명된 값을 속성이라고 함

이름 값 쌍으로 작성된 객체는 다음과 유사함

  • PHP의 연관 배열
  • 파이썬 사전
  • C의 해시 테이블
  • Java의 해시 맵
  • 루비와 펄의 해시

객체 메소드

메서드는 개체에서 수행 할 수 있는 작업임

객체 속성은 프리미티브 값, 다른 객체 및 함수가 될 수 있음

객체 메소드 있는 객체 속성 인 함수 정의

JavaScript 객체는 속성 및 메소드라고 하는 명명된 값의 컨테이너임

JavaScript 객체 만들기

자바스크립트를 사용하면 자신 만의 객체를 정의하고 작성할 수 있음

  • 객체 리터럴을 사용하여 단일 객체를 정의하고 만듬
  • 키워드 new를 사용하여 단일 개체를 정의하고 만듬
  • 개체 생성자를 정의한 다음 생성 된 형식의 개체를 만듬
ECMAScript5에서는 Objext.create()함수를 사용하여 객체를 만들 수 있음

객체 리터럴 사용

자바스크립트 객체를 만드는 가장 쉬운 방법

객체 리터럴을 사용하면 하나의 명령문에서 객체를 정의하고 만들 수 있음

객체 리터럴은 중괄호 {} 안에 name:value쌍의 목록임

Javascript 키워드 사용하기 new

new Object()를 사용할 필요가 없음 단순성, 가독성 및 실행 속도를 위해 첫 번째 (객체 리터럴 메소드)를 사용하십시오

자바스크립트 객체는 변경 가능함

자바스크립트 변수는 변경할 수 없음. 자바스크립트 객체만 가능.
