자바스크립트 번호

자바스크립트에는 한 가지 유형의 번호 만 있음. 숫자는 십진수의 유무와 상관없이 쓸 수 있음. var x = 123e5; var y =123e-5;

숫자는 항상 64비트 부동 소수점임.

정수는 최대 15자리까지 정확

소수점은 최대 수는 17이지만 부동 소수점 산술은 항상 100% 정확하지 않음(문제를 해결하려면 곱셈 및 나눗셈을 수행하는 것이 좋음)

숫자 및 문자열 추가

숫자 + 숫자 = 숫자

문자 + 문자 = 문자

숫자 + 문자 = 문자

문자 + 숫자 = 문자

숫자 문자열

자바스크립트 문자열은 숫자로 된 내용을 가질 수 있음

자바스크립트에서 모든 숫자 연산에서 문자열을 숫자로 변환하려고 시도함

var z = x/y

var z = x*y

var z = x-y

var z = x+y (하지만 이것은 계산 안됨/문자열로 연결시킴)

NaN - 숫자가 아님

NaN은 숫자가 유효한 숫자가 아님을 나타내는 JavaScript 예약어

숫자가 아닌 문자열로 산술 연산을 수행하면 NaN(숫자가 아님)이 뜸.

무한대

Infinity는 가능한 가장 큰 숫자 이외의 숫자를 계산할 때 JavaScript가 반환하는 값

표시할 수 있는 수까지 표현

0으로 나눈 값도 Infinity를 생성

무한대는 number. 무한대는 number를 반환

16진수

자바스크립트는 0x가 앞에 오면 숫자 상수를 16진수로 해석

var x = 0xff; //x will be 255

0과 같이 숫자를 쓰지 마라.(예:07) 슷자를 8진수로 해석함

var myNumber =32;

myNumber.toString(10); //10진수

myNumber.toString(16); //16진수

개체가 될 수 있는 숫자

일반적으로 자바 스크립트 숫자는 리터럴에서 생성 된 프리미티브 값임.

var x=123;

그러나 숫자는 키워드 new를 사용하여 객체로 정의 할 수도 있음.

var y = new(123);

(x==y)와(x===y)의 차이점. 두 개의 자바스크립트 객체를 비교하면 항상 false가 반환.
