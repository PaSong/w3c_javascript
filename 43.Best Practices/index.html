전역 변수 피하기

전역 변수의 사용을 최소화 해라

여기에는 모든 데이터 유형,객체 및 함수가 포함됨

전역 변수 및 함수는 다른 스크립트로 덮어 쓸 수 있음

대신 로컬 변수를 사용하고 클로저 사용 법을 배움

항상 지역 변수 선언

함수에서 사용되는 모든 변수는 지역 변수로 선언해야 함

지역 변수는 var키워드로 선언해야 함. 그렇지 않으면 전역변수가 됨

엄격 모드는 선언되지 않은 변수를 허용하지 않음

상단에 선언

모든 선언은 각 스크립트 또는 함수의 맨 위에 두는 것이 좋은 코딩방법

  • 더 깨끗한 코드 제공
  • 지역 변수를 찾을 수 있는 단일 장소 제공
  • 원하지 않는(함축 된) 전역변수를 쉽게 피해라
  • 원치 않는 재 선언 가능성 감소
    • 기본적으로 JavaScript는 모든 선언을 맨 위로 이동함

변수 초기화

변수를 선언 할 때 변수를 초기화하는것이 좋음

  • 더 깨끗한 코드 제공
  • 변수를 초기화하는 단일 장소 제공
  • 정의되지 않은 값 피하기
변수 초기화는 의도 한 용도(및 의도 된 데이터 유형)에 대한 아이디어를 제공함

숫자,문자열 또는 Boolean 객체 선언 안함

항상 숫자,문자열 또는 부울을 원시 값으로 취급해라.

이러한 유형을 객체로 선언하면 실행 속도가 느려지고 불쾌한 부작용이 발생함

new 객체 사용한함()

new Object() 대신 {}를 사용

new String() 대신 ""

new Number() 대신 0

new Boolean() 대신 false

new Array() 대신 []

new RegExp() 대신 /()/

new Function() 대신 function(){}

자동 유형 변환주의

숫자가 우연희 문자열이나 NaN(숫자가 아님)으로 변환 될 수 있음

수학 연산을 수행 할때 자바스크립트는 숫자를 문자열로 변환 할 수 있음

문자열에서 문자열을 빼면 오류는 발생하지 않지만 NaN(숫자가 아님)을 반환

=== 비교 사용

== 비교 연산자는 항상 비교하기 전에 (일치하는 유형으로) 변환함

===연산자는 값과 유형을 비교함

매개 변수 기본값 사용

누락된 인수로 함수를 호출하면 누락 된 인수의 값이 undefined로 설정됨

정의되지 않은 값은 코드를 손상시킬 수 있음

인수에 기본값을 할당하는 것이 좋은 습관임

스위치를 기본값으로 종료하십시오

항상 스위치 문을 기본값으로 종료해야함. 필요 없다고 생각된데도

eval()사용하지 않기

eval()함수는 텍스트를 코드로 실행하는데 사용됨.

임의 코드가 실행될 수 있기 때문에 보안 문제가 나타남
