Let

ECMAScript 2015

ES2015에서는 let과 const라는 두 가지 중요한 새 JavaScript 키워드를 도입

이 두 키워드는 Block Scope변수(및 상수)를 제공

ES2015이전에는 JavaScript의 범위가 전역 범위 와 함수 범위의 두 가지 유형 밖에 없었음.

글로벌 범위

변수 선언 global(어떤 함수 외부)가 글로벌 범위

var carName ="Volvo"; myFunction(); function myFunction(){ document.getElementById("demo").innerHTML = "I can display" + carName; }

전역 변수는 JavaScript프로그램의 어느 곳에서나 액세스 할 수 있음.

기능 범위

함수 내에서 Locally로 선언 된 변수에는 Function Scope가 있음.

myFunction(); function myFunction(){ var carName = "Volvo"; document.getElementById("demo1").innerHTML = typeof carName + "" + carName; } document.getElementById("demo2").innerHTMl = typeof carName;

지역 변수는 선언 된 함수 내에서만 액세스 할 수 있음

자바 스크립트 블록 범위

var키워드로 선언 된 변수는 블록 범위를 가질 수 없음

{}블록 내에서 선언 된 변수는 블록 외부에서 액세스 할 수 있음

let키워드로 선언 된 변수는 블록 범위르 가질 수 있음.

블록{}안에 선언된 변수는 블록 외부에서 액세스 할 수 없음

{ let x =2; } // x can NOT be used here

변수 재 선언

var키워드를 사용하여 변수를 다시 선언하면 문제가 발생할 수 있음

블록 내에서 변수를 다시 선언하면 해당 변수가 블록 외부에서 재 선언됨.

var x =10; //Here x is 10 { var x = 2; //Here x is 2 } //Here x is 2

let 키워드를 사용하여 변수를 다시 선언하면 이 문제를 해결할 수 있음.

블록 내에서 변수를 다시 선언해도 블록 외부의 변수는 다시 선언되지 않음

var x = 10; //Here x is 10 { let x = 2; //Here x is 2 } //Here x is 10 document.getElementById("demo").innerHTML = x;

브라우저 지원

Internet Explorer11이하에서는 let키워드가 완전히 지원되지 않음

루프 범위

루프에서 var 사용하기:

var i =5; for (var i = 0; i < 10; i ++){ //some ststements } document.getElementById("demo").innerHTML = i; //Here i is 10

루프에서 let사용하기:

let i = 5; for (let i = 0; i < 10; i++){ //some statements } document.getElementById("demo").innerHTMl = i; // Here i is 5

let이 루프에서 i변수를 선언하는 데 사용 되면 i변수는 루프내에서만 볼 수 있음

기능 범위

var 및 let으로 선언 된 변수는 함수 내에서 선언 될 때 매우 유사함

그들은 둘 다 Function Scope:

function myFunction(){ var carName = "Volvo"; //Function Scope }

function myFunction(){ let carName = "Volvo"; //Function Scope }

글로벌 범위

var 및 let으로 선언 된 변수는 블록 외부에서 선언 될 때 매우 유사함

둘 다 Global Scope:

HTMl의 전역 변수

자바스크립트를 사용하면 전역 범위가 자바스크립트 환경임

HTML에서 전역 범위는 윈도우 객체임

var 키워드로 정의 된 전역 변수는 윈도우 객체에 속함

var carName = "Volvo"; //code here can use window.carName document.getElementById("demo").innerHTML = "I can display" + window.carName; // I can display Volvo

let 키워드로 정의된 전역 변수는 윈도우 객체에 속하지 않음

// I cam dosplay undefined

재 선언

var를 사용하여 JavaScript변수를 다시 선언하는 것은 프로그램의 어느 위치에서나 허용됨

동일한 범위 또는 동일한 블록에서 let을 사용하여 var변수를 다시 선언하는 것은 허용되지 않음

var x = 2; // Allowed let x = 3; //Not Allowed { var x = 4; //Allowed let x =5; // Not allowed }

재 선언 하자 가변 하자를 동일한 범위 또는 동일한 블록에서 허용되지 않음.

let x = 2; //Allowed let x =3; //Not allowed { let x =4; //Allowed let x =5; //Not allowed }

let변수를 동일한 범위 또는 동일한 블록에서 var로 다시 선언하는 것은 허용되지 않음

let x = 2; //Allowed var x = 3; //Not allowed { let x =4; //Allowed var x =5; //Not allowed }

let, 다른 범위 또는 다른 블록에서 변수를 다시 선언하는 것은 허용됨

let x = 2; //Allowed { let x = 3; //Allowed } { let x = 4; //Allowed } document.getElementById("demo").innerHTML = x; // 2

호이스팅

var로 전의 된 변수는 맨 위에 올림

선언되지 전에 변수를 사용할 수 있음

carName = "Volvo"; document.getElementById("demo").innerHTML =carName; var carName; //Volvo

let으로 정의 된 변수는 맨위에 올리지 않음

변수가 선언되기 전에 let변수를 사용하면 ReferenceError가 됨.

블록의 시작부터 변수가 선언 될때까지 변수는 "일시적인 불감대"에 있음
