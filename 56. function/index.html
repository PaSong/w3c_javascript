함수정의

함수 선언 또는 함수 표현식을 사용할 수 있음

함수 선언

function functionName(parameters){ //code to be executed }

선언 된 함수는 즉시 실행되지 않음. 그것들은 "나중에 사용되기 위해 저장"되어 나중에 불러낼때 실행

세미콜론은 실행 가능한 JavaScript문을 분리하는데 사용됨 함수 선언은 실행문이 아니기 때문에 세미콜론으로 끝내는 것이 일반적이지 않음

함수 식

자바스크립트 함수는 표현식을 사용하여 정의 할 수도 있음

var x = function (a.b){return a * b}; document.getELementById("demo").innerHTML = x;

함수 표현식을 변수에 저장 한 후에 변수를 함수로 사용할 수 있음

var x = function (a.b){return a * b}; document.getELementById("demo").innerHTML = x( 4 , 3 );

위의 함수는 실제로는 익명함수임

변수에 저장된 함수에는 함수 이름이 필요하지 않음

변수 이름을 사용하여 항상 호출됨

위의 함수는 실행 문에 포함되어 있기 때문에 세미콜론으로 끝남

Function() 생성자

대부분의 경우 new 키워드를 사용하지 않아도 됨

기능 호이스팅

Hoistion은 선언을 현재 범위의 맨 위로 이동시키는 자바스크립트의 기본 동작임

호이스팅은 변수 선언 및 함수 선언에 적용됨

이 때문에 자바스크립트함수는 선언되기 전에 호출 될 수 있음

myFunction(5); function myFunction(y){ return y*y; }

자체 호출 함수

함수 표현식을 자체호출 할 수 있음

자체 호출 표현식은 호출되지 않고 자동으로 호출(시작)됨

식에 ()가 붙으면 함수식으로 자동으로 실행됨

함수 선언을 자체 호출 할 수는 없음

함수 주위에 괄호를 추가하여 함수 표현식임을 나타낼 수 있음

(function () { document.getElementById("demo").innerHTML = "Hello! I called myself"; })(); // 이 예제는 익명함수를 호출한것

함수를 값으로 사용할 수 있음

function myFunction(a, b){ return a * b; } var x = myFunction(4, 3) * 2; document.getElementById("demo").innerHTML = x; // 24

함수는 객체임

typeof JavaScript의 연산자는 함수에 "함수"를 반환

그러나 자바스크립트 함수는 객체로 가장 잘 설명 될 수 있음

자바스크립트 함수에는 속성과 메서드가 있음

arguments.length속성은 함수가 호출 될 때 반은 인수의 개수를 반환

객체의 속성으로 정의 된 함수를 객체에 대한 메소드라고 함 새 오브젝트를 작성하기 위해 설계된 함수를 오브젝트 생성자라고 함

화살표 기능

화살표 함수는 함수 표현식을 작성하기 위한 간단한 구문을 허용함

키워드, 중괄호 및 중괄호function은 필요 없음

//ES5 var x = function(x, y) { return x * y; } //ES6 const x = (x, y) => x * y;

const x = (x, y) => {return x * y}; // 이 방법이 좋은 습관일 수 있음

화살표 기능은 IE11이하에서는 지원되지 않음
