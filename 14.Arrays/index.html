Arrays

자바스크립트 배열은 단일 변수에 여러 값을 저장하는 데 사용

var cars = ["Saab", "Volvo","BMW"];

배열이란?

배열은 한 번에 둘 이상의 값을 저장 할 수 있는 특수 변수

배열 만들기

배열 리터럴을 사용하는 것이 자바스크립트 배열을 만드는 가장 쉬운 방법

var array_name = [item1,item2,...];

공백과 줄바꿈은 중요하지 않음

자바스크립트 키워드 사용하기 new

var cars = new Array("Saab","Volvo","BMW");

new Array를 사용할 필요 없음

배열 요소 액세스

인덱스 번호를 참조하여 배열 요소에 엑세스함.

var name = cars[0]

배열 요소 변경

var cars = ["Saab","Volvo","BMW"]; cars[0] = "Opel"; documnet.getElementById("demo").innnerHTML = cars[0];

전체 배열 엑세스

JavaScript를 사용하면 배열 이름을 참조하여 전체 배열에 액세스 할 수 있음.

배열은 객체임.

배열은 특별한 유형의 객체. JavaScript의 typeOf 연산자는 'object'를 반환

배열 요소가 객체가 될 수 있음.

자바스크립트 변수는 객체가 될 수 있음. 배열은 특별한 종류의 객체

이 때문에 동일한 배열에서 여러 유형의 변수를 가질 수 있음.

배열은 객체를 가질 수 있음. 배열에 함수를 포함 할 수 있음. 배열에 배열을 가질 수 있음.

자바스크립트 배열으 진정한 강점은 내장 배열 속성 및 메서드임.

var x = car.length; // The length property returns the number of elements

\ length 속성은 항상 가장 높은 배열 인덱스보다 하나 더 큼

var y = cars.sort(); // The sort() method sorts arrays

마지막 배열 요소 액세스

var last = fruits[fruits.length -1];

    배열요소 추가하기

    새 요소를 배열에 추가하는 가장 쉬운 방법은 push 메서드를 사용하는 것

    var fruits = ["Banana","Orange","Apple","Mango"]; fruits.push("lemon");

    마지막 요소 뒤에 추가됨.

    length 속성을 사용하여 새 요소를 배열에 추가 할 수 도 있음.

    fruits[fruits.length] = "Lemon";

    높은 인덱스를 가진 요소를 추가하면 배열에 정의되지 않은 '구멍'이 생길 수 있음.

    연관 배열

    많은 프로그래밍 언어는 명명된 인덱스를 사용하여 배열을 지원

    명명된 인덱스가 있는 배열은 연관 배열 (또는 해시)이라고 함.

    자바스크립트는 명명된 인덱스가 있는 배열을 지원 하지 않음.

    자바 스크립트에서 배열은 항상 번호가 매겨진 색인을 사용

    명명 된 인덱스를 사용하면 JavaScript가 배열을 표준 객체로 다시 정의 그런 다음 일부 배열 메서드와 속성은 잘못된 결과를 생성.

    배열과 객체의 차이점

    자바스크립트에서 배열은 번호가 매겨진 색인을 사용

    자바스크립트에서 객체는 명명된 색인을 사용

    배열은 번호가 매겨진 인덱스가 있는 특별한 종류의 객체

    배열을 사용해야하는 경우. 객체 사용시기

    자바스크립트는 연관 배열을 지원하지 않음.

    요소 이름을 문자열(텍스트)로 만드려면 객체를 사용.

    new Array()를 피하라.

    자바스크립트의 기본 제공 배열 생성자인 new Array()를 사용할 필요가 없음.

    대신 []를 사용.

    var points = new Array(); //bad

    var points =[]; //good

    new 키워드는 코드를 복잡하게 함. 예기치 않은 결과를 발생시킴

    배열을 인식하는 방법

    변수가 배열인지 어떻게 알 수 있나??

    해결책1:

    Array.isArray(fruits); //return true

    해결책2:

    자신 만의 isArray()함수를 만들 수 있음.

    인수가 배열이면 우의 함수는 항상 true를 반환

    더 정확하게 : 객체 프로토 타입에 "Array"라는 단어가 포함 되어 있으면 true를 반환

    function isArray(x){ return x.constructor.toString().indexof("Array") > -1; }

    해결책3:

    instanceof 객체가 주어진 생성자에 의해 생성되는 경우 true

    var fruits =["banana","orange","apple","mango"]

    fruits instanceof Array; //return true
