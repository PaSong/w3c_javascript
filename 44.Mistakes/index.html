실수로 할당 연산자 사용

if문의 비교 연산사(==)대신 프로그래머가 실수로 대입 연산자(=)를 사용하면 JavaScript프로그램에서 예기치 않은 결과 가 발생할 수 있음

할당은 항상 할당 값을 반환함

예상되는 느슨한 비교

정기적인 비교에서 데이터 유형은 중요하지 않음. if문은 true를 반환함

switch문이 엄격한 비교를 사용한다는 사실을 잊어 버리는 것은 흔한 실수

혼한스러운 추가 및 연경

추가는 숫자 추가에 관한 것임

연결은 문자열 추가에 관한 것

자바스크립트에서 두 가지 연산 모두 같은 +연산자를 사용함

이 때문에 번호를 숫자로 추가하면 숫자를 문자열로 추가 할 때와 다른 결과가 발생

두 개의 변수를 추가 할 때 결과르 예측하는 것이 어려울 수 있음

오해의 수레

자바스크립트의 모든 숫자는 64비트 부동 소수점 숫자(Float)로 저장됨

자바스크립트를 포함한 모든 프로그래밍 언어는 정확한 부동 소수점 값으로 인해 어려움을 겪음

var x = 0.1; var y = 0.2; var z = x + y; document.getElementById("demo").innerHTML = z; // 0.3000000000004

위 문제를 해결하려면 곱셈및 나눗셈을 수행하는 것이 좋음

var z = (x * 10 + y * 10) / 10; // 0.3

자바스크립트 문자열 깨기

자바스크립트를 사용하면 문장을 두 줄로 나눌 수 있음

문자열에서 명령문을 분리해야하는 경우 "백 슬래시"를 사용해야함

var x = "Hellow" \ "World";

세미콜론의 잘못 배치

잘못 배치된 세미콜론 때문에 코드 블록은 x의 값에 관계없이 실행됨

if(x == 19); { //code block }

반환 진술 깨는

줄의 끝에서 자동으로 문을 닫는 기본 JavaScript 비헤이비어 임

return문을 절대로 사용하지 마십시오

명명된 인덱스를 사용하여 배열에 액세스

많은 프로그래밍 언어는 명명된 인덱스를 사용하여 배열을 지원함

명명된 인덱스가 있는 배열을 연관 배열(또는 해시)이라고 함

자바스크립트는 명명된 인덱스가 있는 배열을 지원하지 않음

자바스크립트에서 배열은 번호가 매겨진 색인을 사용함

자바 스크립트에서 객체는 명명된 색인을 사용함

명명된 인덱스를 사용하면 배열에 엑세스 핦 때 JavaScript가 배열을 표준 객체로 다시 정의함

자동 재정의가 끝나면 배열 메서드와 속성에 의해 정의되지 않거나 잘못된 결과가 생성됨

쉼표로 정의 끝내기

ECMAScript5에서는 객체 및 배열정의의 후행 쉽표가 적합함

points =[40,100,1,5,25,10,];

Internet Explorer8이 중단됨. JSON에서는 쉼표를 사용할 수 없음

정의되지 않은 Null임

자바스크립트 객체,변수,속성 및 메서드는 정의되지 않을 수 있음

또한 빈 자바스크립트 객체의 값은 null일 수 있음

유형이 정의되지 않았는지 테스트하여 객체가 존재하는지 테스트 할 수 있음

document.getElementById("demo").innerHTML = typeof myObj === "undefined";

그러나 객체가 정의되지 않은 경우 오류가 발생하므로 객체가 null인지 테스트 할 수 없음

if (myObj === null) //long

이 문제를 해결하려면 객체가 null이 아니고 정의되지 않았는지(undefined) 테스트 해야함

블록 레벨 범위 예상

자바스크립트는 각 코드 블록에 대해 새 범위를 만들지 않음

많은 프로그래밍 언어에서는 사실 이지만 자바스크립트에서는 사실이 아님
