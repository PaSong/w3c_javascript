Style Guide

모든 자바 스크립트 프로젝트에 항상 동일한 코딩 규칙을 사용

JavaScript코딩규칙

코딩 규칙은 프로그래밍의 스타일 가이드 라인

  • 변수 및 함수의 이름 지정 및 선언 규칙
  • 공백, 들여쓰기 및 설명 사용 규칙
  • 프로그래밍 실습 및 원리

코딩 규칙 안전한 품질

  • 코드 가독성 향상
  • 코드 유지 보수가 쉬워짐

변수 이름

w3schiils에서는 식별자 이름에 camelCase를 사용함

모든 이름은 문자로 시작함

firstName ="John"; price = 19.90;

운영자 주변 공간

연산자 (= + - * / )와 쉼표 다음에 공백을 넣음

var x = y + z;

var values = ["volvo", "saab", "fiat"];

코드 들여 쓰기

코드 블록의 들여 쓰기에는 항상 2칸을 사용함.

들여 쓰기에 탭을 사용하지 마람.다른 편집기는 탭을 다르게 해석함

정규 규정

간단한 문에 대한 일반 규칙

항상 간단한 문장을 세미콜론으로 끝냄

복합명령문에 대한 일반 규칙

  • 첫 번째 줄 끝 부분에 여는 브래킷을 놓음
  • 입구 브래킷 앞에 한 칸을 사용함
  • 공백을 넣지 않고 새 줄에 닫는 괄호를 놓음
  • 복잡한 명령문을 세미콜론으로 끝내지 마람

객체 규칙

객체 정의에 대한 일반 규칙

  • 객체 이름과 같은 줄에 여는 대괄호를 배치함
  • 콜론과 각 속성과 값 사이에 하나의 공백을 사용함
  • 숫자 값이 아닌 문자열 값 주위에 따옴표를 사용함
  • 마지막 속성 값 쌍 뒤에 쉼표를 추가하지 마심
  • 공백을 넣지 않고 새 줄에 닫는 괄호를 놓음
  • 항상 세미콜론으로 객체 정의를 종료하심

라인 길이 80자

가독성을 위해 80자 이상의 행은 사용하지 말아야함

javaScript문이 한 줄에 들어 가지 않으면 가장 좋은 자리는 연산자 또는 쉼표 뒤에 오는 것임

이름 지정 규칙

하이픈은 빼기로 오인 될 수 있음. 자바스크립트에서는 이름에 하이픈을 사용할 수 없음
