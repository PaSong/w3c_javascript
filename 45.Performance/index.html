자바스크립트 코드 속도를 높이는 방법

루프에서 활동 감소

for문을 포함하여 루프의 각 명령문은 루프의 각 반복마다 실행됨

루프 외부에 배치 할 수있는 명령문 또는 지정은 루프를 더 빠르게 실행함

//나쁜예 var i; for(i =0; i < arr.length; i++){}

//더 나은 코드 var i; var l = arr.length; for (i = 0; i < l; i++){}

나쁜 코드는 루프가 반복 될 때마다 배열의 length속성에 엑세스함

더 나은 코드는 루프 외부의 length속성에 액세스하여 루프를 빠르게 실행함

DOM액세스 줄이기

HTML DOM 액세스는 다른 자바스크립트문과 비교할때 매우 느림

DOM요소에 여러번 액세스하려는 경우 한번 액세스하여 로컬 변수로 사용하는게 좋음

var obj; obj = document.getElementById("demo"); obj.innerHTML = "Hellow";

DOM 크기 줄이기

HTML DOM의 요소 수를 작게 유지해라

이렇게하면 항상 페이지로드가 향상되고 특히 작은 장치에서 렌더링 (페이지 표시)이 빨라짐

getElementsByTagName과 같은 DOM검색을 시도할때마다 더 작은 DOM의 이점이 있음

불필요한 변수 피하기

값을 저장하지 않으려는 경우 새 변수를 만들지 마라

var fullName = firstName + "" + lastName; document.getElementById("demo").innerHTML = fullName; 이것으로 바꿀수 있음 document.getElementById("demo").innerHTML = firstName + "" + lastName

지연 자바 스크립트로드 중

스크립트를 페이지 본문의 맨 아래에 두면 브라우저가 페이지를 먼저로드 할 수 있음

스크립트가 다운로드 되는 동안 브라우저는 다른 다운로드를 시작하지 않음

또한 모든 구문 분석 및 렌더링 작업이 차단 될 수 있음

HTTP사양은 브라우저가 두 개 이상의 구성 요소를 동시에 다운로드해서는 안된다는 것을 정의

또한 다른 방법은 스크립트 태그에 defer = "true"를 사용하는 것

defer속성은 페이지가 구문 분석을 마친 후에 스크립트가 실행되도록 지정하지만 외부 스크립트에 대해서만 작동함

가능한 경우 페이지가 로드 된 후 코드별로 스크립트를 페이지에 추가 할 수 있음

함께 사용하지 않기

with키워드를 사용하지 마라.속도에 부정적인 영향을 미침

자바스크립트 범위를 혼란스럽게 함

엄격모드에서는 with 키워드를 사용할 수 없음
