Throw and Try to Catch

  • try : 오류 코드 블록을 테스트 할 수 있음
  • catch : 오류를 처리 할 수 있음
  • throw : 사용자 정의 오류를 만들 수 있음
  • finally : 결과에 관계없이,시도 및 캐치 후, 코드를 실핼 할 수 있음

오류가 발생합니다.

자바스크립트 코드를 실행할 때 다른 오류가 발생할 수 있음

오류는 프로그래머가 작성한 코딩 오류, 잘못된 입력으로 인한 오류 및 기타 예측할 수 없는 것들이 될 수 있음.

try{ adddlert("Welcome guest!"); } catch(err){ document.getElementById("demo").innerHTML =err.message; }

JavaScript는 adddlert를 오류로 처리하고 이를 처리하기 위해 catch코드를 실행

JavaScript try and catch

try문은 당신이 그것을 실행하는 동안 오류를 테스트 할 코드 블록을 정의 할 수 있음.

catch try블록에서 오류가 발생하면 문, 당신은 코드 블록이 실행되도록 정의 할 수 있음.

try 및 catch JavaScript문은 쌍으로 제공됨.

try{ Block of code to try } catch(err){ Block od code to handle errors }

자바 스크립트 오류 발생

오류가 발생하면 JavaScript가 정상적으로 중지되고 오류 메시지가 생성됨.

기술적인 용어는 다음과 같음.JavaScript가 예외를 throw함.(오류발생)

JavaScript는 실제로 name과 message의 두 가지 속성을 가진 Error객체를 만듬.

throw문

throw문을 사용하면 사용자 지정 오류를 만들 수 있음

기술적으로 예외를 throw할 수 있음(오류발생)

예외는 자바 스크립트 문자열,숫자,부울 또는 객체 일 수 있음

throw "Too big"; //throw a text

throw 500; // throw a number

당신이 사용하는 경우 슬로우와 함께 try 및 catch, 당신은 프로그램의 흐름을 제어 및 사용자 지정 오류 메시지를 생성할 수 있음.

입력 검증 예제

입력을 검사함. 값이 틀리면 예외(err)가 발생

예외(err)가 catch문에 의해 catch되고 사용자 정의 오류 메시지가 표시됨

function myFunction(){ var message, x; message = document.getElementById("p01"); message.innerHTML = ""; x = document.getElementById("demo").value; try{ if(x == "") throw "empty"; if(isNaN(x)) throw "not a number"; x = Number(x); if(x < 5) throw "too low"; if(x > 10) throw "too high"; } catch(err){ message.innerHTML = "Input is" + err; } }

마침내 성명

finally문은 결과에 관계없이, 시도 및 캐치 후, 코드를 실행할 수 있음.

try{ Block fo code to try } catch(err){ Block of code to handle errors } finally{ Block of code to be executed regardless of the try / catch result }

오류 개체

JavaScript에는 오류가 발생할 때 오류 정보를 제공하는 오류 개체가 내장되어 있음

오류 개체는 이름과 메시지라는 두가지 유용한 속성을 제공함

오류 개체 속성

이름 : 오류 이름을 설정하거나 반환함

메시지 : 오류 메시지 (문자열)를 설정하거나 반환함

오류 이름 값

오류 이름 등록 정보에서 6개의 다른 값을 리턴 할 수 있음

  • EcalError : eval() 함수에서 오류가 발생
  • RangeError : 숫자 "범위를 벗어났습니다."가 발생
  • ReferenceError : 잘못된 참조가 발생
  • SyntaxError : 구문 오류가 발생
  • TypeError : 유형 오류가 발생
  • URIError encodeURT()오류가 발생

평가 오류

EcalError는 eval()함수에 대한 오류를 나타냄

최신 버전의 JavaScript는 EcalError를 발생시키지 않음. 대신 SyntaxError를 사용

범위 오류

오류 RangeError 당신이 유효한 값의 범위를 벗어나는 숫자를 사용하는 경우 발생

예: 숫자의 유효 자릿수를 500으로 설정할 수 없음

var num = 1; try{ num.toPrecision(500); } catch(err){ document.getElementById("demo").innerHTML = err.name; }

참조 오류

ReferenceError가는 당신이 (참조) 선언되지 않은 변수를 사용하는 경우 발생함

var x; try{ x = y +1; } catch(err){ document.getElementById("demo").innerHTML = err.name; }

구문 오류

구문 에러는 당신이 구문 오류로 코드를 평가하려고 하면 발생

try{ eval("alert('Hello')"); } catch(err){ document.getElementById("demo").innerHTML = err.name; }

유형 오류

형식 오류는 예상 유형의 범위를 벗어나는 값을 사용하는 경우 발생

var num =1; try{ num.toUpperCase(); } catch(err){ document.getElementById("demo").innerHTML =err.name; }

URI(Uniform Resource Identifier)오류

URIError의는 당신이 URI기능에 잘못된 문자를 사용하는 경우 발생

try{ decodeURI("%%%"); } catch(err){ document.getElementById("demo").innerHTML = err.name; }

비표준 오류 개체 속성

Mozilla와 Microsoft는 몇 가지 비표준 오류 객체 속성을 정의

  • fileName 모질라
  • lineNumber 모질라
  • columnNumber 모질라
  • 스택 모질라
  • 설명 마이크로 소프트
  • 번호 마이크로 소프트
